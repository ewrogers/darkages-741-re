# Multi Server (`SMulti`)

| Item | Value |
| --- | --- |
| Direction | Server to client |
| Command | `0x56` (86) |
| Encoding | startup key |
| Name provenance | Microsoft C++ RTTI in the target |
| Behavioral label | Multi Server |

## Purpose

The server sends this message for **multi server**.

The constructor calls `net_server_packet_base_ctor` with opcode `0x56` and installs the exact `SMulti` vtable. `net_server_packet_factory_ctor` registers the same opcode with this constructor.

The multi-server behavior is supported by the client-side [`CMulti`](../client/087-0x57-multi-server.md) path and its `ServerSelectDialogPane` owner. The exact RTTI name remains `SMulti`; it should not be renamed to `SMultiServer` merely to match the friendly label.
