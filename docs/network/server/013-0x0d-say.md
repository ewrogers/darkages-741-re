# Say (`SSay`)

`SSay` puts a temporary speech balloon above a character, creature, or NPC-style world object. It does not save the text in the persistent message history.

| Item | Value |
| --- | --- |
| Direction | Server to client |
| Command | `0x0D` (13) |
| Encoding | session key |
| Name provenance | Microsoft C++ RTTI in the target |

## Body

```text
packet SSay {
    u8 opcode                 // 0x0D
    u8 speech_mode
    u32be object_id
    string8 text
}
```

The text can contain at most 255 bytes because its length prefix is one byte. It remains an ANSI or DBCS byte string rather than Unicode.

## Client behavior

`net_handle_say_server_packet` finds the world object identified by `object_id` and creates its speech balloon for 3,000 milliseconds. The visual or behavioral meanings of the individual `speech_mode` values have not yet been mapped.

This handler never calls `NewSystemMessagePane`. If the server wants the same speech saved, it must also use a history-eligible [`SMessage`](010-0x0a-message.md) type or deliver the text through another history path.

See [Messages and history](../../systems/messages-and-history.md) for the relationship between world balloons, floating notices, and retained text.
