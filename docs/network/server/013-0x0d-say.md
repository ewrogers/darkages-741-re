## Handler and state effect

`net_handle_s_say` at `0x5F3E00` finds `object_id`. A missing object causes `CPutRequest` to be sent. A present object receives a `World::BalloonPane` through `chat_show_object_balloon` at `0x5CBF90`. The balloon is scheduled for removal after `3000` milliseconds.

The local `CSay` input path confirms `say_type` `0` as public say and `1` as shout. The balloon constructor also implements type `2`, but its local sender and semantic name are not yet confirmed.

See [Speech balloons and message routing](../../chat/receive-and-display.md).
