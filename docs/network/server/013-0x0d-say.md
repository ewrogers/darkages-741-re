# Say (`SSay`)

`SSay` attaches public speech to one world object. The server supplies the display mode, sender ID, and text; the client shows it for three seconds.

| Item | Value |
| --- | --- |
| Direction | Server to client |
| Command | `0x0D` (13) |
| Transform | `derived` |
| Concrete class | `SSay` |
| Runtime owner | `WorldPane` and RTTI class `World::BalloonPane` |
| Name provenance | Microsoft C++ RTTI in the target |

## Body

```text
packet SSay {
    u8      opcode                    // 0x0D
    u8      speech_mode               // SpeechMode
    u32     sender_id
    string8 text
}
```

The text can contain at most 255 bytes from its wire length prefix. It is an ANSI or DBCS byte string rather than Unicode.

## Display modes

[`SpeechMode`](../protocol-types.md#speechmode) has three confirmed values:

| Value | Name | Client rendering | Observed server scope |
| ---: | --- | --- | --- |
| `0` | Say | Framed balloon, palette `0xFF`, visually white | Visible range |
| `1` | Shout | Framed balloon, palette `0x45`, visually yellow | Entire map |
| `2` | Chant | Text only above the sender, palette `0x58`, visually blue | Visible range |

The palette indexes are client evidence; final colors come from the active game palette. The scope column is project-owner runtime behavior because recipient selection occurs on the server.

The renderer displays the packet text as supplied. It does not insert an exclamation mark for mode `1`. The local Shout prompt uses `!`, and any `!` present in received shout text must already be in the server-supplied string.

Chant is the display used for timed spell speech. The originating client sends each line through [`CSpellDelaySay`](../client/078-0x4e-spell-delay-say.md), not through interactive `CSay` mode `2`.

## Client flow

`net_handle_say_server_packet` looks up `sender_id`. When the object exists, `ui_world_show_speech` creates `World::BalloonPane` with the selected mode and schedules its removal after 3,000 milliseconds.

When the sender object is absent, the client sends [`CRequestObject`](../client/012-0x0c-request-object.md). It does not queue the current speech for replay after that object arrives.

The client performs no distance or map-scope check after receiving `SSay`. It relies on server delivery and on having the sender object available locally.

This path does not append text to persistent message history. See [Messages and history](../../systems/messages-and-history.md).
