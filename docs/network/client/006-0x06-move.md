# Move (`CMove`)

`CMove` reports one locally accepted cardinal step. The client moves first, then sends the direction with a rolling step byte so the matching [`SMove`](../server/011-0x0b-move.md) reply can confirm the step and measure its round-trip time.

| Item | Value |
| --- | --- |
| Direction | Client to server |
| Command | `0x06` (6) |
| Transform | derived |
| Runtime owner | `WorldPane` |
| Name provenance | Project-owner protocol name, confirmed against the local builder |

## Body

```text
packet CMove {
    u8      opcode                    // 0x06
    u8      direction                 // Direction
    u8      step                      // rolling movement request number
}
```

`direction` uses the shared [`Direction`](../protocol-types.md#direction) type.

## Sending a step

`map_try_move_local_player` checks the destination, moves the local `WorldObject_User`, and calls `net_send_move` only when the step is allowed. The sender then increments the one-byte counter before writing it:

```c
movement_step = movement_step + 1;    // u8 wraparound
build CMove(direction, movement_step);
last_move_send_time = timeGetTime();
submit CMove;
```

The counter starts at zero when `WorldPane` is constructed. The first request therefore carries `1`, and `255` wraps to `0`. The client keeps only one counter and one send time, not a queue of outstanding movement samples.

[`CRefreshUser`](056-0x38-refresh-user.md) does not read, write, or reset either value. The counter continues across a refresh request.

## Four-way movement

The active movement path is cardinal only. Keyboard mapping produces values `0` through `3`, pointer movement divides the world into four directions, and `world_direction_to_delta` defines only those four coordinate changes. No diagonal value reaches `CMove` through these paths.

This does not prove that every trace of an older eight-direction design was removed from the executable. It does confirm that the version 741 movement request, its active input producers, and its coordinate helper implement four directions.

## Reply

The server replies with [`SMove`](../server/011-0x0b-move.md). Its final byte echoes `step`. The client uses a matching echo to update the network indicator and uses the response direction and previous position to advance or correct the world view.
