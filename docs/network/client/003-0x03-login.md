- Direction: client to server

## Current structural notes

- Byte 0 of the untransformed body is opcode `0x03`.
- This client's builder/caller behavior identifies `0x03` as login, even though the related-game enum uses `UserTile` here and moves `Login` to `0x04`.
- The body begins with `u8`-length-prefixed character-name and password strings.
- The builder includes an obfuscated registry-backed `u32` installation identifier and a derived `u16` identifier. The transmitted integer fields use big-endian byte order.
- A 16-byte installation block follows the credentials. It contains two random-byte fields, the encoded identifiers, an encoded 32-bit nonce, a custom 16-bit integrity value over the first 12 block bytes, and trailing bytes `1, 0`.
- Before building the body, the function checks the persistent `SBadGuy` marker flag at client-configuration offset `+0x668`. A set flag triggers `memset(NULL, 0, 100)`, so the login packet is not submitted.
- See [SBadGuy installation marker and login kill switch](../../security/bad-guy-installation-marker.md) for the registry, endpoint, and crash trace.
- See [Client installation identifier](../../security/client-installation-id.md) for the exact assignment, registry packing, XOR masks, and integrity algorithm.
