# Hello (`CHello`)

The client answers the server welcome with the text `baram`. The common submission layer appends a zero byte, then encrypts the resulting six-byte body before it is framed and sent.

| Item | Value |
| --- | --- |
| Direction | Client to server |
| Command | `0x62` (ASCII `b`) |
| Transform | `static` |
| Name provenance | Project-owner protocol name; literal body confirmed in the target |

There is no client packet RTTI class. `CHello` is protocol vocabulary based on its role as the reply to `SHello`. The target itself supplies the older name evidence by formatting the five ASCII bytes `baram`. `baram` may be an engine codename, but the executable does not explain the word.

`net_submit_client_packet` copies those five builder bytes, appends `0x00`, and increases the queued length to six. The outbound transform selector treats the leading `b` as opcode `0x62`, which belongs to the static-key set. Because this happens before `SVersionCheck` installs the negotiated key, it uses the client's compiled startup key.

## Trigger

`TerminalPane2` queues this body when either `ESC C` or `ESC S` completes the initial connection transition. It then calls `net_reset_client_packet_sequence`, builds [Version (`CVersion`)](000-0x00-version.md), and queues that raw body.

## Body

```text
packet CHello {
    u8      opcode                    // 0x62, ASCII "b"
    bytes   text[4]                   // ASCII "aram"
    u8      terminator                // 0x00, appended by net_submit_client_packet
}
```

The confirmed plaintext entering the transform is therefore:

```text
62 61 72 61 6D 00
```

The supplied raw frame matches the client exactly:

```text
AA 00 0E 62 00 34 00 0A 88 6E 23 1E A4 60 7F 4C 6A
|  header | op sq | encrypted payload |  MD5 bytes  | encoded seed |
```

`0x000E` is the 14-byte transformed-body size. The opcode `0x62` and sequence `0x00` remain readable. The five bytes `34 00 0A 88 6E` are the encrypted form of `61 72 61 6D 00`, followed by four selected MD5 bytes and the three-byte encoded seed.

The earlier decoded representation ending in `0xEB` did not reconstruct this packet correctly. The raw frame, submission length, and transform layout agree with one another.

The server trigger is documented under [Hello (`SHello`)](../server/126-0x7e-hello.md).
