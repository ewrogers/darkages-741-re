# Baram (`CBaram`)

| Item | Value |
| --- | --- |
| Direction | Client to server |
| First bytes | `0x62` (ASCII `b`) |
| Encoding | startup key |
| Name provenance | Project-owner protocol name and literal bytes in the target |

## Purpose

The client sends this message for **baram**.

This is a special five-byte control body, not a normal packet class. The client formats the literal ASCII text `baram` and submits exactly five bytes.

The client calls `net_reset_client_packet_sequence` immediately afterward, then builds the next startup body. This ordering makes the observed sequence-reset issues around create, login, exit, and relog especially relevant, but the exact protocol purpose remains unresolved.

## Body

```text
control CBaram {
    u8 text[5]                 // ASCII "baram"
}
```

Do not treat byte `0x62` as sufficient evidence for an ordinary `CBaram` packet layout.
