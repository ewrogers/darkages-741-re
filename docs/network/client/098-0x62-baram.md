# Baram (`CBaram`)

| Field | Value |
| --- | --- |

## Current evidence

This is a special five-byte control body, not a normal reconstructed packet class. At `0x00579832` the client formats the literal ASCII text `baram` and submits exactly five bytes at `0x00579849`.

The client calls `net_reset_client_packet_sequence` immediately afterward at `0x0057993A`, then builds the next startup body. This ordering makes the observed sequence-reset issues around create, login, exit, and relog especially relevant, but the exact protocol purpose remains unresolved.

## Known send sites

- `0x00579849` inside the terminal/startup state machine at `0x00579090`.

## Plaintext body

```text
62 61 72 61 6D             // ASCII "baram"
```

Do not treat byte `0x62` as sufficient evidence for an ordinary `CBaram` packet layout.
